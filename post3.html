1801. Number of Orders in the Backlog

Medium Problem

March 25, 2021 Shiyun Zhang

Problem Analysis

We have a list of orders where orders[i] = [pricei, amounti, orderTypei]. We have two different order types which are ‘BUY’ indicated by 0 and ‘SELL’ indicated by 1. And we always process orders starting from the ith order. Here we know we need to loop through the orders list.

So what’s the rules when we process order:

  1. When the order type is ‘BUY’(0), we need to find the ‘SELL’(1) orders with the smallest selling price, here we know we may need to use the min heap.  Then we need to see if the buying price is greater than or equal to the selling price. If not, then we add the ‘BUY’ order to the backlog. If yes, we will process the ‘BUY’ order:
    • Since the ‘BUY’ order has an amount limit and we need to process the order one by one. We also need to make sure the amount of the ‘SELL’ orders are enough. Here we will use a while loop to process all the ‘BUY’ orders that match with a ‘SELL’ order.
  2. When the order type is ‘SELL’(1), we need to find the ‘BUY’(0) orders with the largesting buying price, here we know we may need to use the max heap.  Then we need to see if the selling price is less than or equal to the buying price. If not, then we add the ‘SELL’ order to the backlog. If yes, we will process the ‘SELL’ order:
    • Since the ‘SELL’ order has an amount limit and we need to process the order one by one. We also need to make sure the amount of the ‘BUY’ orders are enough. Here we will use a while loop to process all the ‘BUY’ orders that match with  a ‘SELL’ order.
